内存管理

发表于 2020-12-10 | 分类于 操作系统 | | 热度: ℃
字数统计: 731 字 | 阅读时长 ≈ 3 分钟

接上篇，回顾 828 的 Lab2，内存管理部分。

从跳转到内核入口点开始

1 控制台初始化

  1. 初始化 cga。
  2. 初始化 kbd。
  3. 初始化 serial。

2 内存初始化

  1. 首先，i386_detect_memory 函数负责探测内存，主要记录两个变量。npages = 32K，npages_basemem = 128 MB。
  2. 然后，创建初始化页目录。
  3. 向页目录中插入页表。
  4. 创建 npages 个 struct PageInfo , 并存储在 pages 数组中。pages 数组中有每一个物理页的入口地址。
  5. 初始化页面。即将空闲的物理页用一个链表连接起来。
  6. 进入内核 monitor 。

详细过程

((void (*)(void)) (ELFHDR->e_entry))();

此指令跳转到 entry.S 中的 _start 指令处执行程序。

1 entry.S 所做工作

  1. 进入内核时，还没有设置虚拟内存，所以程序运行在 1MB 处。但是 C 代码被链接到 KERNBASE + 1MB 处，因此，首先设置一个手写的 4MB 页表。此页表将虚拟地址 [KERNBASE, KERNBASE + 4MB) 翻译到物理地址 [0, 4MB)。此页表一直使用到设置真正的页表为止。

  2. 首先，将 entry_pgdir 的地址（即：手写页表的指针）存入寄存器 cr3。开启分页（即：设置 cr0 中的标志位）。

  3. 跳转到高地址。设置栈指针为 bootstactop，调用 i386_init 。

2 i386_init 初始化

  1. 设置内存大小。

  2. 初始化控制台，包括 cga、kbd 和 serial。

  3. 初始化内存。首先找出机器有多少内存。

    • physical memory : 128 MB
    • npages : 32K

  4. 创建初始页目录 kern_pgdir。 (0xf0345000)

  5. 把页目录作为一张页表插入它自己。

  6. 分配页面。一共 npages 个页面结构，其指针存储在以 pages 为起始指针的数组里。

  7. 页面初始化。即将空闲页面组织为一个链表，以便后续使用。

3 设置虚拟内存

  1. 在 UPAGES 处映射用户只读的页面。（0xEF000000）
  2. 设置内核栈。将 [KSTACKTOP-PTSIZE, KSTACKTOP) 分为两部分： (0xF0000000)
    • [KSTACKTOP-KSTKSIZE, KSTACKTOP) – 内核栈
    • [KSTACKTOP-PTSIZE, KSTACKTOP-KSTKSIZE) – 保护页
  3. 在 KERNBASE 处映射所有物理内存 (256MB 中的 128MB)。（0xF0000000）
  4. 加载 KERNBASE 物理地址进入 cr3，目的是切换到新的页表。（注：cr3 又称 the page directory base register (PDBR)）。
  5. 设置 cr0。开启分页等。
  6. 内存设置完毕，跳转进入montior。
